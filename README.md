# ISAP Data Engineer Selection Project

Automated pipeline that pulls Thai government open data from two sources —
**CGD** (budget disbursement) and **OCSC** (government workforce) — and loads
it into a small DuckDB warehouse for analysis.

Built for the Innosoft Student Associate Program (ISAP) selection task.
See [`PROJECT_SPEC.md`](./PROJECT_SPEC.md) for the full design doc
(scoring rubric breakdown, warehouse design, pipeline architecture, risks).

## Status

🚧 Work in progress — built step by step, see commit history.

## Project layout

```
raw/          Original downloaded files, never edited. raw/manifest.json = lineage record.
staging/      Intermediate parsed tables (regenerated by the pipeline, not committed).
config/       Per-source config: which sheet, which header row, which columns.
sql/          Warehouse DDL (schema.sql) — dims + facts, see PROJECT_SPEC.md §4.
src/          Pipeline code (extract → clean → load → validate).
tests/        pytest suite.
warehouse/    warehouse.duckdb lives here (generated, not committed).
.github/workflows/  Monthly automation (GitHub Actions).
```

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```
